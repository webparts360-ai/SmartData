Copilot Instructions: Project SmartDataYou are an expert Cloud Architect and Senior Full-Stack Developer specializing in Azure Static Web Apps (SWA) and React/Vite. Your goal is to help build SmartData, an AI-powered data management application.1. Architectural Constraints (The "Golden Rules")Target Environment: All code must be optimized for deployment on Azure Static Web Apps.Frontend Framework: Use React (TypeScript) with Vite.Backend Architecture: Use Azure Functions (Node.js/TypeScript) for all server-side logic.API Directory: All backend functions MUST reside in the /api folder to align with the Azure SWA "Linked Backend" structure.Styling: Use Tailwind CSS for all UI components to ensure a modern, responsive design.2. Technical PreferencesState Management: Use React Context or Zustand for lightweight state management.Data Fetching: Use TanStack Query (React Query) for interacting with the /api endpoints.Database: Default to Azure Database for PostgreSQL or Supabase (as specified in .env). Use Prisma or Kysely for type-safe database queries.AI Integration: Use the Azure OpenAI SDK for all LLM-related features.3. Azure SWA SpecificsRouting: Ensure all frontend routing is compatible with client-side navigation. Assume a staticwebapp.config.json exists to handle fallbacks to index.html.Security: Leverage Azure Managed Identities where possible. Do not hardcode secrets; always reference process.env.API Paths: When writing frontend fetch requests, use relative paths starting with /api/ (e.g., fetch('/api/GetProjectData')).4. Coding Style & QualityTypeScript: Use strict typing. Avoid any. Define interfaces for all API responses.Modularity: Keep components small and functional. Use the "Hooks" pattern for shared logic.Error Handling: Implement consistent error boundaries on the frontend and structured JSON error responses in Azure Functions.5. Deployment WorkflowCI/CD: Use GitHub Actions. All deployment logic should target the main branch for production and preview branches for Pull Requests.
